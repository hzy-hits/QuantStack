# quant-research-cn — Design Specification

> This document is the **binding contract** for all development on this project.
> Every module, every function, every output must satisfy these constraints.
> If spec.md and code disagree, spec.md wins.

## 1. System Identity

**quant-cn** is a pure-Rust A-share quantitative research pipeline.

It does ONE thing: compute conditional probabilities from market data, filter by information density, and render a structured Markdown payload for downstream consumption (agent or human).

## 2. Non-Negotiable Constraints

| # | Constraint | Rationale |
|---|-----------|-----------|
| C1 | **No trading signals.** No buy/sell/hold. No position sizing. No execution. | Legal + philosophical: this is research, not a trading system. |
| C2 | **No LLM reasoning.** DeepSeek extracts structured fields from text. It never analyzes, predicts, or reasons. | DeepSeek hallucinates. Extraction is verifiable; reasoning is not. |
| C3 | **Every probability has: horizon, conditioning set, sample size.** | A probability without context is meaningless. `P(up) = 0.62` is useless. `P(5D ret > 0 | trending, low_vol, n=847) = 0.62 [0.58, 0.66]` is useful. |
| C4 | **Agents never touch arithmetic.** All numbers are computed upstream in Rust. | Prevents hallucinated statistics. Agent narrates; program computes. |
| C5 | **Idempotent pipeline.** Running twice on the same date produces identical results. | Deterministic + safe to re-run. |
| C6 | **All data persisted in DuckDB before analytics.** No in-memory-only data paths. | Auditability. Every number traceable to raw data. |

## 3. Mathematical Axiom System

Five axioms govern all analytics. These are not suggestions — every analytics module must trace back to exactly one axiom.

### Axiom 1 — Conditional Probability
```
P(r > 0 | state) ≠ P(r > 0)
```
Regime classification removes non-stationarity. Unconditional return distributions are useless.

- **Module**: `momentum.rs`
- **Implementation**: lag-1 autocorrelation (20 bars) → regime {trending, mean_reverting, noisy} × vol_bucket {low, mid, high} → 9-cell CPT → Beta-Binomial posterior per cell
- **Output**: `P(5D return > 0 | regime, vol_bucket)` with 95% credible interval

### Axiom 2 — Bayesian Updating
```
Prior × Likelihood → Posterior
Beta(α₀, β₀) + (w wins, l losses) → Beta(α₀+w, β₀+l)
```
- **Prior**: Beta(2, 2) everywhere — mildly informative, prevents extreme posteriors on small n
- **Applied in**: momentum, announcement_risk, flow_score, unlock_risk
- **Output**: posterior mean + 95% CI + sample size

### Axiom 3 — Latent States (HMM)
```
Observable returns are generated by hidden market regimes
```
- **Module**: `hmm.rs`
- **Implementation**: 2-state Gaussian HMM on 沪深300 daily returns
  - State 0 (bull): μ > 0, lower σ
  - State 1 (bear): μ < 0, higher σ
- **Output**: P(bull), P(return > 0 tomorrow), regime_duration
- **Calibration**: Brier score on rolling 60-day resolved forecasts

### Axiom 4 — Multi-Source Information Fusion
```
composite = w₁·magnitude + w₂·event + w₃·momentum + w₄·flow + w₅·cross_asset
```
No single signal is sufficient. Orthogonal information sources combine via weighted sum, then macro gate suppresses/amplifies.

| Dimension | Signal Source | A-share Specific |
|-----------|-------------|-----------------|
| magnitude | \|5D return\| z-score | identical to US |
| event | 业绩预告 type → surprise category | replaces US earnings surprise quintile |
| momentum | CPT trend_prob | identical to US |
| flow | 北向 + 融资 + 大宗 + 龙虎榜 + 换手率 | replaces US options (IV/P/C/skew) |
| cross_asset | 行业ETF + 上证50/创业板比值 + 国债 | replaces US sector ETF + VIX + bonds |

### Axiom 5 — Finite Attention
```
Human attention is bounded. Saturate, then filter.
```
- **Module**: `filtering/notable.rs`
- **Implementation**: two-pass filter
  - Pass 1: full universe → rank by composite → top 120
  - Pass 2: 120 candidates + flow/announcement detail → re-rank → top 30
- **Classification**: HIGH (>0.7), MODERATE (0.5–0.7), WATCH (0.3–0.5), LOW (<0.3)

## 4. Signal Definitions

### 4.1 Flow Score (replaces US Options Score)
```
information_score = 0.30 × large_flow_z      # 主力资金净流入 z-score
                  + 0.18 × northbound_z       # 北向资金净买入 z-score
                  + 0.15 × margin_z           # 融资余额变化 z-score
                  + 0.10 × block_z            # 大宗交易溢价 z-score
                  + 0.08 × hot_z              # 龙虎榜净买入 z-score
                  + 0.07 × insider_z          # 股东增减持 z-score
                  + 0.07 × event_clock        # 事件临近度乘数 (1 + 0.35*ev)
                  + 0.05 × market_vol_z       # 全市场波动率 z-score
```
Then blend `tape_abnormality` separately, with iVIX applied as a volatility overlay.

All z-scores clamped to [-3, 3].

### 4.2 Announcement Risk (replaces US Earnings Risk)
```
forecast_type → surprise_category → P(5D excess return > 0 | category)

预增/扭亏     → StrongPositive
略增/续盈     → MildPositive
略减/续亏     → MildNegative
预减/首亏     → StrongNegative
不确定        → Neutral (skip)
```

### 4.3 Unlock Risk (A-share unique)
```
P(5D return < -2% | unlock_ratio, days_to_unlock)

unlock_ratio = shares_unlocking / float_shares
buckets: Small (<1%), Medium (1-5%), Large (>5%)
window: [T-5, T+5] around unlock date
```

### 4.4 Hot Money (A-share unique)
```
signal when: net_buy_ratio > 2σ AND institutional_ratio > 30%

net_buy_ratio = (buy_amount - sell_amount) / circ_mv
institutional_ratio = institutional_buy / total_buy
```

### 4.5 Macro Gate Matrix
3×3 overlay: **上证波动率** × **期限利差 (10Y-1Y国债)**

|  | Calm (<20%) | Elevated (20-35%) | Panic (≥35%) |
|---|---|---|---|
| **Normal spread** | 1.0× | 0.9× | 0.7× |
| **Flat/Inverted** | 0.9× | 0.8× | 0.6× |
| **Steep** | 1.1× | 1.0× | 0.8× |

Per-asset-class multipliers:
- 科技/成长: ×1.2 calm, ×0.7 panic
- 消费/红利: ×0.9 calm, ×1.1 panic
- 周期/资源: follows commodity overlay

### 4.6 Realized Volatility (`rv.rs`)
Three estimators for OHLC data:
- Parkinson: ln(H/L)^2 / (4·ln2) — range-based, 5× more efficient than close-to-close
- Garman-Klass: 0.5·ln(H/L)^2 - (2ln2-1)·ln(C/O)^2 — uses all OHLC
- Yang-Zhang: σ²_overnight + k·σ²_open-to-close + (1-k)·σ²_RS — handles overnight gaps

### 4.7 Volatility Regime (`vol_hmm.rs`)
2-state Gaussian HMM on log(Garman-Klass variance):
- State 0 (`low_vol`): typical calm market
- State 1 (`high_vol`): stress/event-driven regime
- Output: `P(high_vol)`, `P(high_vol tomorrow)`, `vol_regime_duration`
- Used as overlay: `high_vol` regime → reduce conviction on trend signals

### 4.8 iVIX (in `flow.rs`)
VIX-style model-free implied variance from 300ETF options (`OP510300.SH` only):
```
σ²(T) = (2/T) × Σ[ΔK/K² × e^(rT) × Q(K)] - (1/T)(F/K₀ - 1)²
```
Forward inferred via put-call parity. Risk-free rate from Shibor 1M.
Single nearest expiry (T ≥ 7 days). Blended into flow score as vol overlay.

### 4.9 Sector Rotation (`sector_rotation.rs`)
Industry-level momentum and flow aggregation for cross-asset context.

## 6. Pipeline Phases

```
Phase 0: Universe   — load index constituents + watchlist + ETFs
Phase 1: Fetch      — Tushare (7 endpoints) || AKShare bridge (concurrent)
Phase 1.5: Enrich   — DeepSeek structured extraction (async concurrent, Semaphore)
Phase 2: Analytics  — momentum → announcement → flow → unlock → hmm → macro_gate
Phase 3: Filter     — two-pass: universe → 120 → 30 notable items
Phase 4: Render     — Markdown payload → reports/{date}_payload.md
Phase 5: Agent      — Claude reads payload → writes Chinese narrative
Phase 6: Deliver    — Email (optional)
```

## 7. Output Contract

The payload file (`reports/{date}_payload.md`) must contain:

1. **Market Context** — benchmark return, HMM regime, macro gate state
2. **Notable Items** (max 30) — each with:
   - Price action (5D/20D return, volume)
   - Flow data (northbound, margin, block trade signals)
   - Announcements (if any)
   - Probabilities (trend_prob, announcement p_upside, flow_score)
   - Confidence classification (HIGH/MODERATE/WATCH/LOW)
3. **Upcoming Events** — 限售解禁 within 30 days, 业绩预告 deadlines
4. **Cross-Asset Context** — ETF performance, bond yields, commodity signals
5. **Appendix** — methodology notes, sample sizes, disclaimers

## 8. Enrichment Contract (DeepSeek)

DeepSeek is used for **structured extraction only** (constraint C2).

Input: raw news/announcement text
Output: JSON with closed-set fields:
```json
{
  "event_type": "earnings|m_and_a|regulatory|product|lawsuit|management|buyback|dividend|rating|macro|other",
  "sentiment": "positive|negative|neutral",
  "sentiment_confidence": 0.0-1.0,
  "relevance": 0.0-1.0,
  "key_entities": ["公司名", ...],
  "key_metrics": {"metric_name": value, ...},
  "summary_one_line": "一句话摘要 (≤30字)"
}
```

Rules:
- Temperature = 0.0 (deterministic)
- `response_format: json_object`
- System prompt explicitly forbids reasoning/prediction
- Failed parses are logged and skipped (never retry with different prompt)
- Concurrency controlled by `tokio::Semaphore`

## 9. Testing Contract

- `analytics/bayes.rs` — unit tests for Beta-Binomial edge cases
- All analytics modules: known-input → expected-output tests
- HMM: Brier score < 0.25 on 60-day rolling window (or flag degradation)
- Integration: `cargo test` must pass before any commit
