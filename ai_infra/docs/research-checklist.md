# AI Infra 研究判断基线 / Checklist

## 使用目的

这份 checklist 用来判断一个 AI Infra 产业链环节或公司，是否只是 AI 叙事映射，还是具备真实需求、真实瓶颈、真实议价权和长期增长弹性。

使用对象：

- 单个产业链模块，例如 HBM、CPO、先进封装、NeoCloud、电力设备。
- 单家公司，例如 NVDA、AVGO、MU、COHR、TOWA、Soitec、IQE、BE、LEU。
- 一个跨公司主题，例如“存储超级周期是否是 AI Infra 指数增长起点”。

核心原则：

- 先证明需求，再讨论弹性。
- 先找反证，再给叙事。
- 先看公司原始披露，再看二手摘要。
- 先看财报、订单、产能和客户证据，再看新闻稿。
- 先判断是否处在 AI 大基建周期，再讨论静态估值倍数。
- 先判断产业链位置，再判断估值空间。

## 0. 原始出处校验规则

凡是涉及实际数据，先找原文，再写结论。这里的实际数据包括但不限于：

- 收入、segment revenue、AI/data center revenue、客户集中度。
- backlog、RPO、订单、长约、预付款、产能锁定。
- ASP、bit shipment、wafer start、良率、交期、出货量。
- gross margin、operating margin、CapEx、inventory、free cash flow。
- 数据中心电力容量、机柜功率、液冷渗透率、网络端口速率。

证据优先级：

1. 公司年报、10-K、20-F、quarterly report、earnings release。
2. earnings call transcript、investor presentation、capital markets day。
3. 交易所公告、监管文件、公司官网技术白皮书和产品规格。
4. 产业协会、标准组织、客户或供应商的交叉披露。
5. 媒体、券商、数据库和 ChatGPT Pro 输出只能作为线索。

原文校验时必须记录：

- 来源链接、发布日期、报告期、公司主体和证券代码。
- 指标口径：GAAP/non-GAAP、segment 定义、单位、币种、同比/环比。
- 原文能直接证明什么，不能证明什么。
- 由原文推导出的判断，必须单独标记为“推论”。
- 如果只有二手来源，结论状态标记为“待原文核验”，不得进入核心判断。

## 1. 一句话研究假设

每个研究对象都必须先写清楚一句话假设：

```text
如果 LLM 训练/推理需求持续增长，那么 [模块/公司] 会因为 [瓶颈/议价权/技术壁垒] 获得 [收入/毛利率/现金流/估值] 的持续提升。
```

例子：

```text
如果 AI 推理集群继续扩大，那么 HBM 和服务器 DRAM 会因为带宽和容量瓶颈，获得 ASP、出货量和毛利率的持续提升。
```

如果一句话假设写不清楚，先不要进入公司研究。

## 2. 产业链位置判断

| 检查项 | 要问的问题 | 证据 |
| --- | --- | --- |
| 需求距离 | 它离 LLM 训练/推理有几层？优先使用 `D0-D5 BFS 深度`。 | 客户、订单、产品用途 |
| 价值链位置 | 是核心瓶颈、二阶供应商、配套设备，还是主题映射？ | BOM、供应链关系、客户披露 |
| 是否可替代 | 客户能否快速换供应商或换技术路线？ | 认证周期、良率、生态锁定 |
| 是否被平台内化 | 云厂商或芯片龙头是否会自研替代？ | 自研 ASIC、内部供应链、并购 |

### LLM Dependency BFS 深度

本项目优先使用从 LLM Lab 出发的 BFS 深度，而不是泛泛的“AI 相关”标签。

| BFS 深度 | 定义 | 研究权重 |
| --- | --- | --- |
| `D0` | LLM 核心需求源头：OpenAI、Anthropic、Google DeepMind、Meta、xAI 等 | 最高，但很多未上市 |
| `D1` | 模型强依赖一阶：GPU/TPU/ASIC、云算力、训练/推理软件栈 | 最高 |
| `D2` | 一阶依赖的直接瓶颈：HBM、CoWoS、网络、光互连、AI server/rack、电力冷却 | 很高 |
| `D3` | 二阶瓶颈供应商：测试、设备、基板、光子、EDA/IP、连接、材料、液冷组件 | 高弹性重点区 |
| `D4` | 深层制造与材料：化学品、气体、真空、洁净、衬底、精密加工 | 雷达区 |
| `D5` | 基础资源与约束：能源、电网、土地、水、融资、监管 | 远端约束区 |

研究范围规则：

- 重点研究 `D1-D3`。
- `D0` 用来定义需求，不一定容易找到公开资产。
- `D4-D5` 最深看到即可，默认只做雷达；只有能证明反向卡住 `D0-D2` 时才升级。
- `D6+` 默认排除。

进入下一层必须有边关系：客户边、BOM 边、产能边、技术边、现金流边或反证边。仅靠“AI 概念”不能进入主图。

原有辅助分层标签：

- `S0 需求源头`：LLM 模型、AI 应用、API、企业工作流。
- `S1 核心算力`：GPU、TPU、ASIC、AI 加速器。
- `S2 直接瓶颈`：HBM、先进封装、网络、数据中心电力。
- `S3 配套扩张`：服务器、液冷、光模块、交换芯片、测试设备。
- `S4 上游制造`：设备、EDA、材料、气体、化学品、基板。
- `S5 主题映射`：收入关联弱，主要靠市场叙事。

研究优先级通常是 `S2 > S1 > S3 > S4 > S0 > S5`，但 10 倍弹性常出现在 S3/S4 小盘瓶颈中。

## 3. 需求真实性

| 检查项 | 强证据 | 弱证据 |
| --- | --- | --- |
| 客户订单 | 长单、预付款、backlog、产能锁定 | 新闻稿、合作备忘录 |
| 收入体现 | AI 相关收入明确增长 | 管理层口头提 AI |
| 客户结构 | Hyperscaler、GPU 厂、HBM 厂、封测厂直接采购 | “潜在客户很多” |
| 价格信号 | ASP 上升、合同价上调、供不应求 | 股价上涨 |
| 产能信号 | 扩产、设备订单、交期拉长 | 市场传闻 |

必须回答：

- 需求来自谁？
- 客户为什么必须买？
- 这个需求是训练、推理、数据中心建设，还是普通半导体周期？
- 订单是否已经进入收入，还是仍在 pipeline？
- 是否存在库存重建或补库存误判？

红旗：

- 公司只说“AI opportunity”，不披露订单、客户、产品或收入。
- 收入增长来自消费电子、汽车、工业恢复，却被包装成 AI。
- 客户集中在单一大客户，但没有长期合同。

## 4. 供给瓶颈

| 检查项 | 要问的问题 |
| --- | --- |
| 扩产周期 | 新产能需要几个月、几年，还是十年以上？ |
| 技术门槛 | 难点是良率、材料、设备、软件生态，还是资本支出？ |
| 认证周期 | 客户认证是否很长，切换供应商是否困难？ |
| 关键设备 | 是否依赖 ASML、TEL、AMAT、Lam、KLA、TOWA、DISCO 等设备？ |
| 地域限制 | 是否受出口管制、地缘风险、电网许可影响？ |

瓶颈强度分级：

- `A 真瓶颈`：产能短期无法复制，客户愿意预订多年，例如高端 HBM、先进封装、部分电力设备。
- `B 中瓶颈`：扩产需要时间，但竞争者较多，例如部分光模块、服务器集成。
- `C 周期瓶颈`：当前紧缺，但供给可快速扩张，例如部分通用材料或普通 NAND。
- `D 伪瓶颈`：主要来自叙事，不是客户真正卡点。

## 5. 议价权与利润池

| 检查项 | 要问的问题 |
| --- | --- |
| 毛利率 | 需求上升时毛利率是否同步提升？ |
| ASP | 公司是否能涨价，还是只能出更多货？ |
| 客户议价 | 客户是 NVIDIA/Apple/云厂商这类强势买家吗？ |
| 竞争格局 | 是寡头、双寡头，还是大量同质竞争？ |
| 差异化 | 产品靠技术、认证、良率、软件生态，还是靠低价？ |

判断：

- 收入涨、毛利率也涨：较强议价权。
- 收入涨、毛利率不涨：可能只是代工/集成/低附加值。
- 收入涨、现金流恶化：可能被 CapEx 或库存吞掉。
- ASP 涨、出货也涨：强周期或结构性瓶颈。

## 6. 持续性

| 时间维度 | 要验证的问题 |
| --- | --- |
| 6-12 个月 | 是否只是补库存、抢货、短缺？ |
| 1-3 年 | 是否有客户 CapEx、订单、产能规划支撑？ |
| 3-5 年 | 技术路线是否仍需要该环节？ |
| 5-10 年 | 是否受益于 LLM 使用量和推理量指数增长？ |

核心问题：

- 这是一次性 CapEx，还是持续折旧后还要追加？
- 推理效率提升会减少需求，还是降低成本后扩大总需求？
- 云厂商是否会过度建设，导致下一轮价格战？
- 开源模型和小模型是否会削弱云端大模型需求？

## 7. 财务传导

| 财务项 | 看什么 |
| --- | --- |
| Revenue | AI 相关收入占比、增长率、客户集中 |
| Gross margin | 产品 mix、ASP、良率、规模效应 |
| Operating margin | 研发、销售费用是否吞掉毛利 |
| CapEx | 扩产是否带来未来收入，还是造成折旧压力 |
| Inventory | 是否提前备货导致后续减值 |
| Backlog / RPO | 订单是否可见，是否可取消 |
| FCF | 收入增长是否转化为自由现金流 |
| ROIC | 扩产回报率是否高于资本成本 |

最低要求：

- 不能只看收入增长。
- 必须看毛利率和 FCF。
- 对重资产公司必须看折旧、库存、CapEx 回收周期。
- 对云和 NeoCloud 必须看利用率、折旧、融资成本、客户期限。

## 8. 技术替代风险

| 风险 | 示例 |
| --- | --- |
| ASIC 替代 | TPU、Trainium、Maia 分流 NVIDIA GPU |
| 软件优化 | vLLM、FlashAttention、量化降低硬件需求 |
| 架构变化 | MoE、long context、KV cache 改变内存/网络需求 |
| 光电路线切换 | CPO、硅光、LPO、可插拔光模块路线变化 |
| 先进封装路线 | CoWoS、SoIC、hybrid bonding、panel-level packaging |
| 能源路线 | 核电、天然气、燃料电池、储能、PPA 组合变化 |

必须回答：

- 什么技术变化会让这个环节需求下降？
- 什么技术变化会让这个环节需求反而上升？
- 公司是否有能力跟随路线切换？

## 9. 周期与结构性拆分

| 类型 | 特征 | 处理方式 |
| --- | --- | --- |
| 结构性增长 | 多年订单、供给瓶颈、客户持续扩张 | 可以进入核心研究池 |
| 周期反转 | ASP 从低点修复，库存正常化 | 只按周期估值，警惕高点 |
| 主题映射 | AI 叙事强，财报弱 | 放入观察池，不进入核心 |
| 一次性订单 | 单客户大单，没有复购证据 | 需要折价处理 |

存储超级周期要拆成：

- HBM：最接近 AI 真实瓶颈。
- 服务器 DRAM：较强 AI 推理和云扩张关联。
- eSSD：数据中心和 AI 数据管线相关，但要区分普通企业存储恢复。
- NAND：可能既有 AI，也有传统周期，必须谨慎。
- 控制器/测试/封装材料：可能有小盘弹性，但需要客户证据。

## 10. 基建周期、资产重估与预期差

AI super cycle 不能只用传统华尔街静态 PE 框架判断。PE 是结果指标，不是研究起点。先判断它是否处在跨时代基础设施扩张中，再判断市场是否已经过度定价。

| 检查项 | 要问的问题 |
| --- | --- |
| 基建属性 | 该环节是在重建计算、电力、网络、存储、制造基础设施，还是普通周期恢复？ |
| CapEx 持续性 | 客户未来 3-5 年是否仍要持续建设，而不是一次性补库存？ |
| 瓶颈租金 | 供给紧缺是否能转化为 ASP、毛利率、预付款或长约？ |
| 利润池迁移 | 旧 IT/半导体利润池是否向该环节迁移？ |
| 市场是否已经知道 | 这是 consensus trade 还是 early discovery？ |
| 定价是否透支 | 当前估值隐含几年增长？是否已经把完美执行计入价格？ |
| 业绩弹性 | 小幅收入增长能否带来大幅 EPS/FCF 增长？ |
| 下修风险 | 如果 AI CapEx 放缓，估值会如何重估？ |
| 流动性 | 小盘是否容易因为流动性造成过度波动？ |

估值使用方式：

- 不用静态 PE 直接否定早期基础设施周期。
- 不用低 PE 直接证明公司便宜；先确认它是否真在 AI Infra 利润池中。
- 对重资产建设者，更重视 CapEx 回收周期、利用率、融资成本和 FCF 拐点。
- 对瓶颈供应商，更重视 ASP、毛利率、长约、客户认证和扩产约束。
- 对小盘弹性标的，更重视收入基数、AI 相关收入占比、客户证据和增量利润率。

10 倍候选通常需要：

- 起始市值不大。
- AI 相关收入占比正在从低到高。
- 毛利率有上行空间。
- 能在新基建周期中获得瓶颈租金或利润池迁移。
- 市场尚未完全理解其产业链位置。
- 有真实瓶颈或客户认证壁垒。
- 未来 3-5 年收入和利润可能重估。

大盘核心资产更可能是稳定复利，不一定容易 10 倍。

## 11. 反证清单

每个主题必须写反证：

- 如果客户 CapEx 放缓，这个环节会不会最先被砍？
- 如果 NVIDIA 供给放开，是否会压缩该环节利润？
- 如果 HBM 供给过剩，存储周期是否反转？
- 如果云厂商 ASIC 成功，GPU 生态哪些环节受损？
- 如果推理成本下降 10 倍，总硬件需求是下降还是上升？
- 如果电力并网卡住，哪些设备先受益，哪些订单延迟？
- 如果客户换供应商，公司护城河是否存在？
- 如果毛利率不提升，原来的瓶颈假设是否失败？

## 12. 评分表

每家公司或模块按 1-5 分打分：

| 维度 | 1 分 | 5 分 |
| --- | --- | --- |
| AI 需求相关度 | 弱叙事 | 直接被 LLM 训练/推理拉动 |
| 供给瓶颈 | 易扩产 | 多年瓶颈 |
| 议价权 | 低毛利、客户强势 | 可涨价、毛利率上行 |
| 持续性 | 1 年周期 | 5 年以上结构性 |
| 财务传导 | 收入不进利润 | FCF 和 ROIC 改善 |
| 技术护城河 | 易替代 | 高认证、高良率、高生态锁定 |
| 基建周期位置 | 普通周期或弱映射 | 新计算/电力/网络/制造底座 |
| 资产重估空间 | 已充分定价 | 市场低估或未覆盖 |
| 反证清晰度 | 证据弱 | 可跟踪、可证伪 |

分类：

- `40+`：核心研究池。
- `32-39`：重点观察池。
- `24-31`：主题跟踪池。
- `<24`：暂不投入深挖。

## 13. 标准研究流程

1. 写一句话假设。
2. 判断它是否属于 AI 大基建周期，而不是普通景气轮动。
3. 先列出需要核验的原始披露来源。
4. 确定产业链位置。
5. 找到真实客户和订单证据。
6. 判断供给瓶颈和扩产周期。
7. 看毛利率、ASP、backlog、CapEx、FCF。
8. 列技术替代和周期反转风险。
9. 打分并放入四类池子。
10. 写下一轮验证问题。
11. 用财报和产业资料复核。
12. 每季度更新一次判断。
